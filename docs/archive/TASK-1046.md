### Context

635 tests are flat test() calls. 3 test files use describe blocks. Files with 30+ tests (drift-checker.test.ts) have no visual hierarchy to distinguish unrelated check groups. Three naming conventions coexist ("behavior first", "fixture - behavior", "Subsystem - behavior"). This is a readability issue, not a correctness issue — test output is harder to scan when a failure appears among 30 peer-level tests with no grouping. TASK-1043 (mock consolidation) first — once test setup is standardized, reorganization is easier. Enforce the chosen naming convention via a note in a test style guide, not via a lint rule (overkill for this codebase size).
