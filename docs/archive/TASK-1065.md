### Context

`core.md §2` states: "Every commit must reference a TASK-ID (except `idea:` drafts)."

`arch analyze` and `arch govern` produce commits tagged `[THINK]` or `[GOVERN]` with no associated task — they are system-mode commits, not implementation work. The rule as written technically prohibits them. This was surfaced as a rule gap in downstream project smartcart-os.
