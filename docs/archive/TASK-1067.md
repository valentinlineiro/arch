### Context

`arch task start TASK-XXX` injects context by querying the corpus for semantically similar past tasks. When the corpus contains only governance artifacts (new project, first sprint), every query returns 0.00 confidence. The human receives no useful context.

The task meta line already contains a context path (e.g. `**Meta:** P1 | M | READY | Focus:no | 2-code-generation | claude-code | src/server/routes/`). This path is used today to scope `git log` evidence but is not used to answer: "what are the most-changed functions in this directory recently, and what do those files look like?"

Extend `arch task start` context injection with a codebase phase that runs before (or instead of, when corpus confidence is low) the corpus query: identify candidate files from the context path, surface recently changed functions via git log/diff, and inject as `[CODEBASE-CONTEXT]` suggestion.
