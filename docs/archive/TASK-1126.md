### Context

MISSION.md is a document agents read and interpret. Two different models can read "new collectors following existing pattern" and classify the same IDEA differently. That is probabilistic, not deterministic.

The fix: move the classification lists out of prose and into arch.config.json as a closed enum. MISSION.md becomes human-readable documentation of those enums, not the source of truth.

arch.config.json:
```json
"mission": {
  "autonomousScope": ["bug-fix", "new-collector", "scoring-tuning", "documentation", "test-coverage"],
  "humanGated": ["schema-change", "new-api-endpoint", "new-dependency", "scoring-architecture", "output-format-change"]
}
```

arch review can now determine autonomy classification by: read Mission-class from IDEA file, look up in config array, return autonomous | human-gated | unclassified. No model needed.
