### Context

RETRO.md records what happened. It does not tell a fresh model what ARCH is right now.

The multi-model design problem: each session reads the codebase cold, forms a local model, and extends it. When those models differ, the codebase accumulates contradictions (CausalGraph, UEG layer, dead domain services).

The fix: at sprint close, RETRO.md gets a ## Current State section written in present tense. What ARCH is now. What was removed and why. What the active commands are. What the active subsystems are. What the open strategic questions are.

A model reading RETRO.md at session start should form the same mental model as the model that wrote it. That alignment is the mechanism that prevents multi-model drift.
