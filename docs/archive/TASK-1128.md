### Context

arch analyze --scan currently emits IDEAs with no Mission-class and Decision: Pending human review. The MISSION.md document is read but not parsed. Classification requires a human or model to read the IDEA and decide.

Replace with structural classification: each detector assigns Mission-class based on file paths, extensions, and content patterns.

Classification rules (deterministic):
- File in collectors/ + same function signature → "new-collector"
- File in migrations/ or *.sql modified → "schema-change"
- package.json dependencies changed → "new-dependency"
- Route pattern added to index.ts (new URL path) → "new-api-endpoint"
- File in tests/ only → "test-coverage"
- Only *.md files changed → "documentation"
- Scoring weights/thresholds changed in score*.ts → "scoring-tuning"
- Ungoverned file in src/ → "bug-fix" (default for ungoverned code)
- Forward action not matching above → "documentation" (default)

Decision field set automatically: autonomousScope → "Pending human review.", humanGated → "AWAITING_HUMAN — <class> requires human confirmation per arch.config.json mission block"
