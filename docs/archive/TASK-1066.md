### Context

A PROMOTED IDEA's `Decision: PROMOTE → TASK-XXX` field can reference the wrong task ID with no automated check. Concrete instance found in downstream project smartcart-os: IDEA-auth referenced TASK-018 which was an onboarding fix, not auth-related. The IDEA appeared resolved but wasn't.

The fix: add a written protocol under THINK Phase 1 requiring cross-check of the referenced task title against the IDEA intent. Mismatches surface as a Data Integrity Alert in INBOX.

Source: IDEA-user-issue-reporting (Finding 2).
