### Context

Without a human intent file, an autonomous loop promotes its own IDEAs and executes them. It optimises for what it can complete easily, not what matters. The closed loop drifts from intent silently.

MISSION.md defines: what this repo is trying to accomplish, what classes of change are in-scope for autonomous execution, and what requires human approval before promotion.

Structure:
```markdown
# MISSION.md
## Goal
One sentence. What this repo produces and for whom.

## Autonomous scope
Classes of work the loop can execute without human confirmation:
- bug fixes with failing tests
- dependency updates
- documentation improvements
- refactors with no behavior change

## Human-gated
Classes of work that require human Decision field before promotion:
- new features
- schema changes
- API surface changes
- external integrations

## Non-goals
What this repo explicitly does not do.
```

arch analyze --scan checks IDEA candidate-class against MISSION.md before emitting. If the IDEA class is human-gated, it marks Decision: AWAITING_HUMAN instead of leaving it as Pending.
