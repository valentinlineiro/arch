**Decision:** ResumeCommand implemented. 6 halt→recovery paths: REVIEW_FAIL (reset to READY + ledger entry + commit), INTEGRITY_BREACH (run govern), FOCUS_VIOLATION (run govern), BUDGET_EXHAUSTED (prompt new size), CORPUS_HALT (run corpus audit, no close), Unknown (print context). **Constraint:** REVIEW_FAIL path resets task to READY but does not re-run the Auditor review — human must re-review after reset. This is intentional: the Auditor determines if the task was the problem, not arch resume.
**Cost:** No architectural debt. BUDGET_EXHAUSTED interactive prompt requires stdin — will not work in non-interactive CI. Acceptable for a recovery command.
