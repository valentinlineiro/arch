### Context

ARCH generates IDEAs two ways: THINK.md Phase 2 (retrospective) and manual capture. Both are reactive. The brownfield adoption of market-intel exposed a gap: ARCH can see the codebase, but has no mechanism to generate IDEAs from what it observes before the human notices friction.

Demonstrated signal: the StackOverflow collector (TASK-003) was added manually, and the human noticed the 5-step wiring friction after the fact. `IDEA-easy-source-addition` was filed manually. A codebase scanner would have surfaced it after the second collector was added.

`arch analyze --scan` (or `arch analyze --brownfield`) adds a prospective IDEA generation mode. It scans the codebase for patterns that signal missing abstraction, ungoverned history, and unresolved forward actions. Output: IDEA files in `docs/refinement/` with `Source: codebase-scan` and specific evidence.

**Decision:** arch analyze --scan implemented with 3 detectors: (1) boilerplate pattern detection — finds 3+ files sharing identical function signature shape; (2) ungoverned file detection — git log scan for files modified 3+ times with no TASK-ID in commit messages; (3) forward action mining — scans last 100 archived tasks for non-placeholder Forward Actions not yet promoted to IDEAs. Deduplication against existing refinement queue. Common files (README.md, package.json) exempted. Forward action cap at 10/scan. Tested on market-intel (6 IDEAs) and ARCH (12 IDEAs). 634 tests pass.
**Constraint:** Boilerplate detector uses function signature shape normalization — may produce false positives on highly generic signatures (string, void). Forward action threshold at 40 chars reduces noise but may miss short actionable entries.
**Cost:** analyze --scan adds ~2-5 seconds on a large repo (git log + readdir scans). Acceptable as an on-demand command.
**Forward Action:** Add arch.config.json governance.scanExemptions list for repos that want to suppress specific detectors.
