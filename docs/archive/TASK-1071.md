### Context

When `arch govern` moves a task from REVIEW → DONE → archive, the corresponding `REVIEW_REQUEST` block in `docs/INBOX.md` is left behind. The INBOX continues showing "Pending review: TASK-XXX" even though the task has been archived.

This was observed in the SmartCart pilot: TASK-032 was archived by govern tick 33, but its REVIEW_REQUEST remains in `smartcart-os/docs/INBOX.md`.

Already documented in ARCH's own `docs/KAIZEN-LOG.md`:

> **Friction:** After 23 tasks completed in a single day, the INBOX still showed "READY tasks: 5", two REVIEW_REQUEST entries for tasks already DONE in archive.
> **Root Cause:** INBOX written by DO sessions but never invalidated by `arch govern`.
> **Proposed Fix:** `arch govern` should strip resolved REVIEW_REQUEST and AWAITING_PROMOTION entries from INBOX as part of its archiving sweep.
