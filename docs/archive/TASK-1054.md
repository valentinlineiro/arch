### Context

Signal from valentinlineiro/smartcart-os: TASK-001 was started (`arch task start` sets Focus:yes) then abandoned. Govern preserved its focus indefinitely via the inercia window — blocking TASK-003 (real P1 work) from getting FOCUS_ACQUIRED until TASK-001 was manually force-closed.

Root cause: `arch task start` sets `Focus:yes` but there is no staleness detection. A task started and abandoned holds focus forever. The init template is correct (Focus:no) — the issue is that starting then abandoning any task creates a stuck state.

Fix: govern detects tasks that have been IN_PROGRESS for longer than N ticks (configurable, default 10) with no commits to the task's context paths. Surfaces a `[STALE_TASK]` alert in INBOX and yields focus to the next eligible task.
