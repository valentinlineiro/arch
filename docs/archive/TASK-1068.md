### Context

ARCH verifies task-level correctness (AC predicates pass → task closes). It does not verify system-level correctness (primary user flows still work after N tasks close). In the smartcart-os pilot, TASK-008 correctly stubbed POST /api/inventory per the cut list; the main CTA called that endpoint, silently breaking basket entry. No govern-level check covered end-to-end flow integrity.

Add a `## Core Flows` section to `PROJECT.md` with runnable predicates. `arch govern` checks all Core Flows predicates on every tick. `arch task review` emits advisory `[FLOW-REGRESSION]` warnings when a core flow is failing and context path overlaps. Govern output includes a Core Flows health summary.
