### Context

`arch analyze` iterates over `clis` from config but calls `process.exit(0)` after the first CLI runs — success or failure. If the primary CLI returns non-zero (quota exhausted, network error), the error prints but execution never falls through to the next CLI. opencode is already registered in `arch.config.json` with a valid template. The fix is ~5 lines: check `result.status`, continue loop on non-zero, only exit on success. Source: IDEA-opencode-fallback. Novelty: LOW (TASK-183/210/921 all implement provider fallback).
