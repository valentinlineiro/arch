### Context

archProfile: minimal controls enforcement (Hansei is advisory, no corpus, no sprint). It does not yet control the visible command surface. A user running arch help on a minimal repo still sees all 14 commands including audit, corpus, triage, sentinel, serve.

This is noise for a new user. The visible surface should match the profile. On minimal: show capture, start, done, govern, review, status, fix, init, upgrade. On standard: add analyze, triage, ask. On full: all commands.

The profile already controls behavior — it should also control visibility. A command that does nothing useful in minimal profile should not appear in arch help for a minimal repo.
