### Context

In `cli/src/test/ts/meta-line-parser.test.ts`, four malformed-input tests use `assert.ok(task === null || task)`. This assertion is always true: `task === null` covers the null case; `task` covers the truthy case. The expression can never be false. The tests only verify that the parser doesn't throw — they don't verify it returns `null`. Discovered during TASK-1044 Auditor review.

Lines affected: 159, 165, 174, 181.

The two final malformed tests (lines 187, 192) correctly use `assert.strictEqual(task, null)`. The fix is to determine the parser's actual behavior on each bad input and assert that specific value.
