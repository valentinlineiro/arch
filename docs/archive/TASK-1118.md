### Context

arch init currently prompts for profile: [M]inimal / [S]tandard / [F]ull. Default behavior when non-interactive is unknown. The pedagogically correct default is minimal — show the core loop, let the work demand more.

A new user should experience ARCH as: capture a task, start it, do the work, close it with a brief Hansei, run govern. No sprint warnings, no corpus federation, no 34 drift checks. Just the loop. If they hit a problem the loop doesn't solve, they upgrade.

This is the difference between opt-in and opt-out complexity. Right now ARCH defaults to full enforcement and lets you turn things off. It should default to minimal and let you turn things on.
