## TASK-1035: move process.exit() to main entry point: commands return exit codes
**Meta:** P0 | M | DONE | Focus:yes | 2-code-generation | local | cli/src/main/ts/index.ts, cli/src/main/ts/application/commands/
**Closed-at:** 2026-05-26T13:07:25.769Z

**Depends:** TASK-1033

### Context

At least 14 command files call `process.exit()` directly. This makes commands untestable (process.exit() terminates the test runner), prevents the dispatcher from handling errors uniformly, and makes it impossible to compose commands. The fix is standard: commands return an exit code (number) or throw a typed error; `index.ts` calls `process.exit()` once, on the result. `TASK-1033` (command routing into registry) is a prerequisite because it clarifies the dispatcher's contract before this task changes what commands return.
