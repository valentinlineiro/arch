## TASK-1077: Phase-aware sprint close: sprintCloseOn milestone predicate
**Meta:** P2 | S | READY | Focus:no | 1-code-reasoning | claude-code | cli/src/main/ts/application/use-cases/govern-system.ts, arch.config.json

**Depends:** none

### Context

Source: IDEA-phase-aware-sprint-close.md

arch govern auto-closes a sprint when archivedThisTick >= sprintCloseAfterN. For ARCH-building-ARCH, count and completion correlate. For product sprints (smartcart-os), a sprint closed mid-implementation because 15 tasks happened to archive: definition, research, and scaffolding tasks: while 7 implementation tasks were still open.

Add sprintCloseOn: "milestone" to arch.config.json. When set, govern closes the sprint only when a named milestone predicate in PROJECT.md ## Core Flows passes, not when a task count is reached.

### Acceptance Criteria

- [ ] When `sprintCloseOn: "milestone"` is set in arch.config.json, govern does not auto-close on task count.
  - `prose: 15+ tasks archived with sprintCloseOn:milestone set → sprint remains open`

- [ ] Govern auto-closes when the milestone predicate in PROJECT.md passes.
  - `prose: sprint closes when all Core Flow predicates pass with sprintCloseOn:milestone`

- [ ] Default behavior (sprintCloseAfterN) unchanged when sprintCloseOn is absent.
  - `prose: existing repos without sprintCloseOn still close on count`

- [ ] `npm test` passes.
  - `prose: 707+ tests pass`
