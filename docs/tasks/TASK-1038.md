## TASK-1038: domain service consolidation: audit and archive speculative services
**Meta:** P3 | M | READY | Focus:no | 3-chore | local | cli/src/main/ts/domain/services/

**Depends:** none

### Context

`domain/services/` has 56 files. Several are speculative subsystems added during ontology-first development that are not called from any active code path: transient-actuation-projector.ts, stateless-hypothesis-generator.ts, transient-window-compiler.ts, measurement-governance-layer.ts, action-governance-service.ts, decision-observability-service.ts, institutional-anomaly-tracker.ts. These inflate the domain model surface, make context indexing noisier, and create maintenance overhead for imports and types that nothing uses. The task is an audit pass, not a redesign: confirm dead code, delete or archive confirmed dead services, document any that are intentionally kept despite having no current callers.
