## TASK-1030: Archive partitioning strategy - year/sprint subdirectories in docs/archive/
**Meta:** P3 | S | READY | Focus:yes | 6-writing | local | docs/archive/

**Depends:** none

### Context

The docs/archive/ directory is a flat namespace. As it grows, file count and listing latency increase. This task scopes the partitioning strategy — year/sprint subdirectories — and defers execution until the archive crosses 500 files (trigger condition).

**Trigger:** promote when docs/archive/ exceeds 500 files.

### Acceptance Criteria

- [ ] Design the partitioning scheme: `docs/archive/YYYY/sprint-slug/` vs `docs/archive/YYYY-MM/` or similar.
- [ ] Document the scheme in `docs/guidelines/archive.md`.
- [ ] Write an ADR for the partitioning decision (ADR-XXX). Include rollback procedure.
- [ ] Write a migration script (`scripts/migrate-archive.sh`) that re-partitions existing files.
- [ ] Update `arch.config.json` paths to reflect new archive structure.
- [ ] Update `arch review` archive integrity checks to support partitioned paths.

### Definition of Done
- [ ] Partitioning scheme approved via ADR
- [ ] Migration script tested against a copy of the archive
- [ ] All references to flat `docs/archive/` updated
