## TASK-1069: automatic corpus federation on govern tick
**Meta:** P2 | M | READY | Focus:no | 2-code-generation | local | cli/src/main/ts/infrastructure/

### Context

ARCH's corpus learning is project-local by default. Inter-project federation is supported (`arch corpus import`) but manual. Manual means it doesn't happen. The Hansei from pilot projects contained genuine domain learning but lived only in the local archive, invisible to other projects. The compounding value of inter-project learning was never realized.

Add `corpus.remotes` to `arch.config.json` with `syncOnGovern: true`. On each govern tick, run a delta sync: fetch remote archive manifest, download only changed Hansei digests, merge into local corpus index. Sync is non-blocking — if remote is unreachable, govern logs a warning and continues. Local archive tasks are also auto-indexed on govern without requiring manual `arch corpus import`.

### Gaps

None identified — scope is bounded to config schema, delta sync logic, and auto-indexing on govern.
