## TASK-1010: arch init: minimal project bootstrap for external repos
**Meta:** P1 | M | READY | Focus:no | 2-code-generation | claude-code | cli/src/main/ts/application/commands/, arch.config.json
**Depends:** none
**Created-at:** 2026-05-25T00:00:00.000Z

### Context
ARCH currently only works inside its own repo — paths, corpus, and protocol files are all assumed present. To be testable as a product, it must work on a fresh external repo from a single command: `arch init`.

This task implements the `arch init` command and the minimal starter corpus it generates.

### Acceptance Criteria
- [ ] `arch init` command exists and is wired into the CLI dispatcher
  - `file: cli/src/main/ts/application/commands/`
- [ ] Running `arch init` in an empty directory creates exactly: `arch.config.json` (minimal), `docs/tasks/` directory, `docs/INBOX.md`, and one READY starter task (`TASK-001: Complete your first governed task`)
  - `cmd: arch init --dry-run; exit: 0`
- [ ] The generated `arch.config.json` is valid and passes `arch review` on the new repo
  - `cmd: arch review; exit: 0`
- [ ] The starter task walks the user through `arch task start → arch review → arch task done` in its description
  - `file: docs/tasks/TASK-001.md`
- [ ] `arch init` is idempotent: running it twice does not overwrite existing files
- [ ] `npm test` passes
  - `cmd: npm test --prefix cli; exit: 0`

### Definition of Done
- [ ] All ACs verified.
- [ ] `arch review` passes.
