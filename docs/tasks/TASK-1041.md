## TASK-1041: integration test: arch check against real temp git repo
**Meta:** P1 | M | READY | Focus:no | 2-code-generation | local | cli/src/test/ts/

**Depends:** TASK-1039

### Context

40/69 test files use MockFileSystem. MockFileSystem records reads and writes but does not enforce real filesystem semantics: path resolution, symlinks, permissions, git object model. For a tool that writes to .arch/, executes git commands via SubprocessRunner, and has a pre-commit hook that blocks commits, the mock-vs-reality gap is a correctness risk, not a style issue. The single highest-value test the suite doesn't have: run `arch check` against a real (temp) git repo, assert on stdout. This catches what mocks can't — cases where the real git or real FS would behave differently. One well-chosen integration test is worth more than all the describe() organization in the moderate issues list.
