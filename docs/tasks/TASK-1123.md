## TASK-1123: arch review --env: environment pre-flight check before task start
**Meta:** P1 | S | READY | Focus:no | 1-code-reasoning | local | cli/src/main/ts/application/commands/review-command.ts

**Depends:** none

### Context

If the environment is broken (npm install failed, git conflict, test suite broken) the autonomous loop proceeds into broken state. Every task it runs will fail. Without distinguishing "task is wrong" from "environment is broken", the loop retries endlessly.

arch review --env runs before arch task start in autonomous mode. Checks:
1. git status clean (no uncommitted conflicts)
2. npm test passes (or configured test command)
3. No ANDON_HALT entries in NOTIFICATIONS.md
4. arch review passes (no blocking drift)

If any check fails, exit non-zero with a diagnostic. The loop orchestrator treats non-zero as "halt until human clears".

### Acceptance Criteria

- [ ] arch review --env checks: git status clean, test suite passes, no ANDON_HALT in NOTIFICATIONS.md, arch review passes
  - `cmd: node cli/dist/index.js review --env; exit: 0`
- [ ] arch review --env exits non-zero with a clear diagnostic when any check fails
  - `prose: broken test suite causes arch review --env to exit 1 with "tests failing: N failures" message`
- [ ] arch task start runs arch review --env automatically in autonomous mode (when ARCH_AUTONOMOUS=1 env var is set)
  - `prose: ARCH_AUTONOMOUS=1 arch task start TASK-XXX runs --env check first and aborts if it fails`
- [ ] npm test passes
  - `prose: 634+ tests pass`
