## Hansei
**Severity:** H1
**Category:** [ToolingError]
**Decision:** A `sed -i` command introduced broken single-quoted template strings (`'${this.pr.archDir}/context/...'`) in build-index.ts — TypeScript does not interpolate `${}` in single-quoted strings. The error was caught on context restore and corrected with targeted `replace_all` edits before the build. No broken code shipped to dist.
**Constraint:** The sed approach was used to batch-replace a large number of occurrences across a file. The correct approach for multi-occurrence replacements in TypeScript files is `replace_all` with exact string matches, not shell sed with regex — sed cannot distinguish quote types.
**Cost:** ~20 minutes of correction work across context boundary.
**Forward Action:** Avoid `sed -i` for TypeScript string replacements. Use Edit tool replace_all exclusively for bulk substitutions in .ts files.
