## TASK-1027: progressive CLI surface — two-tier help visibility
**Meta:** P0 | M | READY | Focus:no | 2-code-generation | local | cli/src/main/ts/application/command-dispatcher.ts, cli/src/main/ts/domain/services/command-registry.ts

**Depends:** none

### Context

`arch help` currently exposes ~30 commands at first contact. The depth that makes ARCH powerful for experienced users is an acquisition wall for new users. This task restructures the help surface into two visibility tiers: a default surface (3 commands) and a full surface (`--full` flag). No state machine, no usage tracking, no tier progression — only a visibility gate. All commands remain callable at all times; only what appears in `arch help` is changed. Source: IDEA-progressive-cli-surface (Decision: simplified from three-tier state machine to two-surface gate).

### Acceptance Criteria

- [ ] `arch help` (default, no flags) displays exactly three commands: `init`, `review`, `task capture`, with one-line descriptions each.
  - `prose: arch help output contains only init, review, task capture — no other commands listed`

- [ ] `arch help --full` displays all available commands in the existing format.
  - `prose: arch help --full shows the same command inventory as current arch help`

- [ ] All commands not in the default surface remain fully callable (e.g., `arch govern` works even though it is not in default help).
