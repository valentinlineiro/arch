## TASK-1079: Context injector planned artifact projection fix
**Meta:** P2 | S | READY | Focus:no | 1-code-reasoning | claude-code | cli/src/main/ts/application/use-cases/build-index.ts

**Depends:** none

### Context

Source: IDEA-context-injector-planned-artifact-projection.md

The context injector (BuildIndex) extracts artifact references from task description prose and surfaces them in the context index. When a task says "This task will produce ADR-001", the injector includes ADR-001 as a referenced artifact: even though it doesn't exist yet. Downstream agents read the context and treat ADR-001 as authoritative, causing hallucination and incorrect assumptions.

Fix: filter artifact references to only those that exist on disk at index time.

### Acceptance Criteria

- [ ] When a task mentions an artifact name in prose that does not exist on disk, the injector does NOT include it in the context index.
  - `prose: task mentioning non-existent ADR-042 does not add ADR-042 to context index`

- [ ] Existing artifacts referenced in task prose ARE included as before.
  - `prose: task mentioning ADR-001 (which exists) still appears in context index`

- [ ] `npm test` passes.
  - `prose: 707+ tests pass`
