## TASK-1042: remove as any from test mocks: implement typed mock classes
**Meta:** P1 | S | READY | Focus:yes | 2-code-generation | local | cli/src/test/ts/

**Depends:** TASK-1039

### Context

Multiple test files use anonymous objects cast as any to satisfy interface requirements (e.g. `{ getAll: async () => [] } as any`). When a refactor changes the interface, the cast compiles silently and only fails at runtime. TypeScript strict mode exists to catch exactly this — as any in mocks disables it at the point where it matters most. The fix: replace anonymous any-cast objects with proper mock classes that explicitly implement the relevant interfaces. Any missing methods become compile errors, which is the desired behavior.
