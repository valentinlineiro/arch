## TASK-1025: arch corpus import: federated corpus from external ARCH repos
**Meta:** P2 | S | READY | Focus:yes | 1-code-reasoning | claude-code | cli/src/main/ts/application/commands/corpus-import-command.ts, cli/src/main/ts/application/use-cases/corpus-index.ts, cli/src/main/ts/application/command-dispatcher.ts

**Depends:** none

### Context

ARCH corpus is siloed per repo. Patterns from an external project (grocery optimizer, PGW) are invisible to the main corpus. `arch ask` can't surface cross-project signals. Federated import enables compounding across projects. Source: IDEA-corpus-import. Import is one-way and read-only — no state merging.

### Acceptance Criteria

- [ ] `arch corpus import <path>` reads another ARCH repo's `docs/archive/TASK-*.md` and `docs/adr/ADR-*.md` and merges them into the local corpus index with a `source` tag derived from the repo directory name.
  - `file: cli/src/main/ts/application/commands/corpus-import-command.ts`

- [ ] `arch corpus import https://github.com/x/y` clones the repo to a temp dir, imports, then cleans up. Same outcome as path import.
  - `prose: verified by running arch corpus import https://github.com/valentinlineiro/arch — entries tagged source:arch`

- [ ] `arch corpus import <path> --as <slug>` uses explicit slug instead of directory name.
  - `prose: arch corpus import /path/to/grocery-optimizer --as grocer → entries tagged source:grocer`

- [ ] Imported entries appear in `arch corpus audit` output with their source tag.
  - `prose: arch corpus audit shows imported entries with source: grocer`

- [ ] `arch ask` accepts `--project <slug>` to filter query to one source.
  - `prose: arch ask --project grocer "product matching" returns only grocer entries`

- [ ] Import does NOT copy task files to `docs/tasks/`, does NOT merge `.arch/` state.
  - `prose: docs/tasks/ and .arch/ unchanged after import`

- [ ] Registered as `arch corpus import` in command dispatcher.
  - `file: cli/src/main/ts/application/command-dispatcher.ts`

- [ ] `npm test` passes.
  - `prose: 544+ tests pass`

- [ ] `arch review` passes.
  - `cmd: node cli/dist/index.js review`
