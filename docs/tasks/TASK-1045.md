## TASK-1045: coverage thresholds in CI: fail below 60% branch coverage
**Meta:** P1 | M | REVIEW | Focus:yes | 3-chore | local | cli/package.json, .github/workflows/

**Depends:** TASK-1040, TASK-1041

### Context

Coverage reporting (TASK-1040) measures; this task enforces. Without a threshold, coverage is informational only and will drift down under sprint pressure. 60% branch coverage is a conservative starting threshold for a governance tool — it can be raised once the baseline from TASK-1040 is known. TASK-1041 (integration tests) must complete first so the threshold is set against a suite that includes real-FS tests, not just mocks. The CI configuration exists (.github/workflows/) but does not currently run tests — this task also wires up the test run.

### Acceptance Criteria

- [x] `.github/workflows/` contains a CI job that runs `npm test` and fails the build on test failure.

  - `prose: test.yml runs npm test then npm run test:coverage — fails on test failure`

- [x] `npm run test:coverage` is run in CI and fails if branch coverage drops below 60%.
  - `prose: c8 config in cli/package.json sets branches:60, lines:60, functions:60, statements:60`

- [x] The threshold is documented with a comment explaining it is a floor, not a target.

- [x] CI job completes in under 3 minutes on a cold runner.
