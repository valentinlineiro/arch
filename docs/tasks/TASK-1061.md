### Context

Promoted from IDEA-context-injector-planned-artifact-projection.

The context injector extracts artifact references (ADR names, file paths) from task description prose and includes them in the context index. When a task *mentions* an artifact by name — e.g. "This task will produce ADR-001" — the injector surfaces that artifact as if it already exists. Observed in smartcart-os: "ADR-001: Cost-only optimization for Cádiz pilot _(advisory)_" appeared in task context before ADR-001.md existed.

A task agent reading that context may confidently reference ADR-001 as an authoritative constraint before it has been written. The `_(advisory)_` marker partially mitigates this, but the entry still appears in context — misleading enough to cause drift.

Fix: after extracting artifact references from task text, resolve each ADR/file reference against the filesystem. Only include entries where the artifact file exists. References to planned-but-not-yet-created artifacts are omitted entirely (or emitted with a distinct `[PLANNED]` marker that agents are explicitly instructed not to treat as authoritative).
