### Context

Building a project with ARCH requires manually writing the first ADR and decomposing the spec into tasks. This bottleneck prevents autonomous project initialization. Source: IDEA-project-scaffold. Advisory only — LLM proposes, human ratifies before execution.

### Acceptance Criteria

- [ ] `arch project init "<spec>"` command exists and is registered in dispatcher.
  - `file: cli/src/main/ts/application/commands/project-command.ts`

- [ ] Command calls the primary configured LLM CLI with a structured decomposition prompt. Prompt instructs LLM to output JSON: `{ adrs: [{title, context, decision}], tasks: [{title, class, size, depends, acs}], dod: [string] }`.
  - `prose: verified by running arch project init "todo app" and inspecting output`

- [ ] Generated ADRs written to `docs/adr/ADR-NNN-*.md` (auto-numbered after last ADR). Generated tasks written to `docs/tasks/TASK-NNN.md` using standard task format with all required fields.
  - `prose: verified by inspecting generated files — valid format, arch check passes`

- [ ] `docs/PROJECT.md` created with: goal, scope, generated DoD predicates (from LLM output), and `## Ratification` section requiring human sign-off before loop can start.

  - `prose: verified by disabling all CLIs — clean error, no partial output`

  - `prose: arch project init "todo app" --depth 1 generates ~5 tasks; --depth 2 generates ~15`
