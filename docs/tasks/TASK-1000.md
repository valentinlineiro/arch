**Depends:** none

### Context

ARCH is only adoptable from greenfield. Almost all real repos are brownfield. Phase 1 of brownfield-onboarding: `arch audit --public <url>` runs against any public GitHub repo without install, producing a structural MRI. No ADRs generated. No governance. Observations only — explicitly labeled as such to avoid the "auto-generating convincing garbage" failure mode. Source: IDEA-brownfield-onboarding-pipeline (Phase 1 only).

### Acceptance Criteria

- [ ] `arch audit --public https://github.com/x/y` clones the repo, runs structural analysis, and outputs a terminal-readable MRI without requiring ARCH to be installed in the target repo.

- [ ] Output includes: module count, hotspot candidates (high churn files), coupling indicators (files changed together), orphan candidates (files with no recent commits), and a summary line ("N hotspots, M orphan files, K% hidden coupling").
