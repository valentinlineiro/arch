### Context

Promoted from IDEA-arch-release-and-cd.

ARCH is published to npm manually today. This task implements the full CD loop:

1. **Sprint close → auto-tag**: `govern-system.ts` sprint-close path bumps the patch version in `cli/package.json` + `arch.config.json`, creates a git tag `vX.Y.Z`, and pushes it.
2. **Tag → publish**: new `.github/workflows/release.yml` triggers on `v*.*.*` tags — builds, tests, publishes to npm, creates a GitHub Release with a `feat:`/`fix:` changelog.
3. **Consumer upgrade**: `arch upgrade` (TASK-1055) uses the published version as its source.

Version bump strategy: patch on every sprint close. Minor/major requires `nextVersionBump: minor|major` in `arch.config.json` (govern reads it, applies it once, resets to `patch`).
