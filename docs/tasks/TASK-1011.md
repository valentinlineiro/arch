### Context
The ARCH CLI has hardcoded paths scattered across 52 files (215 occurrences). Any external repo using ARCH via npm must adopt this exact directory structure. This 3-phase decoupling makes all paths configurable via `arch.config.json`.

Phase 1 (this task): create `PathResolver` — a typed service that reads `arch.config.json` once and exposes all path constants with defaults. No consumer wiring yet. This is the foundation Phases 2 and 3 build on.
Phase 2 (TASK-1014): wire PathResolver into the core execution layer (govern-system, task-repository, loop-engine, focus-ledger, escalation-store).
Phase 3 (TASK-1015): wire PathResolver into commands and analysis layer (drift-checker, build-index, init-command, and the long tail).
