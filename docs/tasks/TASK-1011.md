### Context
The ARCH CLI has hardcoded paths (e.g. `docs/tasks/`, `docs/archive/`, `docs/INBOX.md`, `.arch/`) scattered throughout the codebase. Any external repo using ARCH via npm must adopt this exact directory structure. Phase 1 of decoupling: make all paths configurable via `arch.config.json` so `arch init` can emit a minimal config and the CLI honours it.

Phase 2 (rules engine plugin model, full protocol schema) is deferred — that is an architectural refactor requiring an ADR. This task is the prerequisite: get paths out of source code.
