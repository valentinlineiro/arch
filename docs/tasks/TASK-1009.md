## TASK-1009: INBOX hygiene: automatic stale entry cleanup in govern tick

**Depends:** none

### Context

`docs/INBOX.md` grows without expiry. Stale AWAITING_REVIEW for closed tasks, duplicate PATTERN-ALERTs, 14-day-old advisory entries. Humans stop reading it. Source: IDEA-inbox-hygiene.

### Acceptance Criteria

- [ ] `govern-system.ts` runs INBOX hygiene after each tick: removes AWAITING_REVIEW/AWAITING_PROMOTION lines whose task/IDEA is in `docs/archive/`.
  - `prose: verified by checking INBOX after govern — closed task entries removed`

- [ ] Duplicate PATTERN-ALERT for same category: keeps only most recent.
  - `prose: two [PATTERN-ALERT] [SpecDrift] entries → one after govern`

- [ ] Non-deterministic entries (`[ADVISORY]`, `[SEMANTIC-DRIFT]`, `[THINK]`) older than 14 days removed. Configurable via `governance.inboxExpiryDays` in arch.config.json (default 14).
  - `prose: verified by injecting a 15-day-old [ADVISORY] entry — removed after govern`

- [ ] `ANDON_HALT`, `CORPUS_ALERT`, `PATTERN-ALERT` entries never auto-removed by age.
  - `prose: deterministic alerts persist across ticks regardless of age`

- [ ] `arch govern --clean-inbox` forces a full hygiene pass.
  - `cmd: node cli/dist/index.js govern --clean-inbox; exit: 0`

- [ ] `npm test` passes.
  - `prose: 544+ tests pass`
