### Context
The escalation store accumulated 71+ phantom OPEN entries because the lifecycle has an open end: THINK appends AWAITING_PROMOTION on IDEA surfacing, but nothing appends RESOLVED at decision time. The deduplication fix (TASK-999) prevents new duplicates on append but does not close the loop. Discovered during manual backlog review — 4 sessions of cleanup work that should be automated.

Two-part fix: (A) protocol — require RESOLVED write when a Decision is written; (B) structural — DriftChecker cross-check catches any that slip through.
