If Gemini returns non-zero (quota exhausted, 429), the error prints but execution never falls through to opencode. opencode is already registered in `arch.config.json` with a valid template. The fix is ~5 lines: check `result.status`, continue loop on non-zero, only exit on success. Source: IDEA-opencode-fallback. Novelty: LOW (TASK-183/210/921 all implement provider fallback).
