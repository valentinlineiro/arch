## TASK-1005: Decouple arch govern reflect from govern tick : enforce structural LLM/governance separation
**Meta:** P1 | S | READY | Focus:yes | 1-code-reasoning | claude-code | cli/src/main/ts/application/use-cases/govern-system.ts, cli/src/main/ts/application/commands/reflect-command.ts

**Depends:** none

### Context

ADR-026 (Epistemic Layer Separation) declares `arch govern` deterministic. But `govern-system.ts` currently calls `AuditCommand.runQuiet()` and hooks into `arch govern reflect` in the reflect path, creating a structural seam where LLM output (from `arch analyze`) can flow back into govern's execution cycle.

The principle: governance gates must be owned by deterministic code. Advisory is fine but must be structurally isolated — no path from LLM output to a governance pass/fail decision.

Currently broken:
1. `arch govern reflect` runs inside the govern tick on deep cadence — LLM advisory output flows into INBOX.md which govern reads for PATTERN-ALERT signals
2. INBOX entries from THINK are mixed with deterministic alerts — no structural distinction
3. An agent in CI reading exit codes cannot distinguish "govern halted on deterministic ANDON" from "reflect wrote a warning"

### Acceptance Criteria

- [ ] `arch govern reflect` is removed from the govern tick. Reflect runs only when explicitly invoked (`arch govern reflect` as a manual command or separate scheduled job). The govern tick never calls reflect or analyze.
  - `prose: verified by reading govern-system.ts — no reflect/analyze invocation`

- [ ] `arch govern` tick is fully deterministic: no LLM invocations, no shell-out to advisory channels, no reading of reflect output as input.
  - `prose: verified by reading govern-system.ts execute() method — no spawnSync on any LLM binary`

- [ ] INBOX.md entries from THINK/reflect are written with a `[ADVISORY]` prefix; deterministic entries (ANDON_HALT, PATTERN-ALERT, CORPUS_ALERT) retain their current prefixes. The distinction is structural, not just visual.
  - `grep: "\\[ADVISORY\\]" cli/src/main/ts/application/commands/reflect-command.ts`

- [ ] `arch govern` exit code is always determined by deterministic checks only. ANDON_HALT from a deterministic check = non-zero. Advisory output = always 0 govern exit.
  - `prose: verified by running arch govern with broken reflect config — govern still exits 0`

- [ ] ADR-034 filed extending ADR-026 with explicit govern/reflect separation rule.
  - `file: docs/adr/ADR-034-govern-reflect-separation.md`

- [ ] `npm test` passes.
  - `prose: 544+ tests pass`

- [ ] `arch review` passes.
