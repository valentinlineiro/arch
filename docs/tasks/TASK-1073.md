## TASK-1073: arch triage: resolve AWAITING_TRIAGE staged IDEAs from remote corpus
**Meta:** P2 | S | READY | Focus:no | 1-code-reasoning | claude-code | cli/src/main/ts/application/commands/triage-command.ts, cli/src/main/ts/application/command-dispatcher.ts, .arch/incoming-ideas/

**Depends:** TASK-1072

### Context

TASK-1072 wired incoming remote IDEAs to `.arch/incoming-ideas/` and INBOX `[AWAITING_TRIAGE]` entries. But there is no command to resolve them. A human currently has to: read the staged file, decide PROMOTE/NOISE/DEFER, manually delete the file from `.arch/incoming-ideas/`, and manually remove the INBOX entry. That is four error-prone steps. `arch triage` collapses them into one interactive flow.

### Acceptance Criteria

- [ ] `arch triage` lists all staged IDEA files from `.arch/incoming-ideas/` with a one-line summary each (IDEA title + source slug + status).
  - `file: cli/src/main/ts/application/commands/triage-command.ts`

- [ ] `arch triage <filename>` opens the full IDEA content and prompts for a decision: `[P]romote / [N]oise / [D]efer`.
  - `prose: running arch triage smartcart-IDEA-user-*.md shows content and prompts`

- [ ] On **Promote**: copies IDEA file to `docs/refinement/<filename-without-slug-prefix>.md` (entering ARCH refinement queue), removes staged file, removes `[AWAITING_TRIAGE]` INBOX entry.
  - `prose: after Promote, file appears in docs/refinement/ and INBOX entry is removed`

- [ ] On **Noise**: removes staged file and INBOX entry. No copy to refinement.
  - `prose: after Noise, staged file deleted and INBOX entry removed`

- [ ] On **Defer**: leaves staged file and INBOX entry in place, appends `Deferred: YYYY-MM-DD` line to the staged IDEA file so it's visible on next triage.
  - `prose: after Defer, staged file still present with Deferred date appended`

- [ ] `arch triage --all` runs triage for every staged file sequentially without re-listing.
  - `prose: arch triage --all prompts for each staged IDEA in turn`

- [ ] `arch triage` with no staged IDEAs prints "No incoming IDEAs to triage." and exits 0.
  - `prose: on empty .arch/incoming-ideas/ arch triage exits cleanly`

- [ ] Registered in CommandDispatcher.
  - `file: cli/src/main/ts/application/command-dispatcher.ts`

- [ ] `arch review` passes.
  - `cmd: node cli/dist/index.js review`
