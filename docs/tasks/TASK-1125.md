## TASK-1125: Hansei verification separation: separate session rule in DO.md
**Meta:** P2 | S | READY | Focus:no | 6-writing | local | docs/agents/DO.md, docs/agents/THINK.md

**Depends:** none

### Context

The agent that implements a task writes the Hansei in the same session with full implementation context. It knows exactly what it built. It has an incentive (efficiency) to write "no deviation occurred." The Hansei degrades into a validation ceremony.

The fix is a protocol rule, not code: Hansei must be written in a separate arch analyze session from implementation. The analyzing agent receives only: the task file, the test output, and the git diff. No implementation context. It cannot know what the implementing agent intended — it can only observe what actually happened.

This creates genuine separation between implementation and retrospective. The Hansei becomes an independent audit rather than a self-assessment.
