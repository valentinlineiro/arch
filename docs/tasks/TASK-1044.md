### Context

The meta line (`**Meta:** P0 | S | READY | Focus:no | 2-code-generation | local | docs/tasks/`) is the source of truth for task state in ARCH. `MarkdownTaskRepository.parseTask` parses it; serialization writes it back. If parsing is wrong, governance is wrong — the wrong task status propagates through every govern tick. Current tests verify known-good inputs. They don't verify the parser is a correct inverse of the serializer, and they don't verify behavior on boundary or malformed inputs. A round-trip property test (`parse(serialize(task)) === task`) catches parser regressions that unit tests miss because it doesn't depend on the author anticipating the specific failure mode.
