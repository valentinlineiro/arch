### Context

`arch status` reads `**Integrity Level**` from `docs/METRICS.md`, which is only fully regenerated by `arch report`. After a gap between `arch report` runs, the Integrity Level can be days old with no indication of staleness — users see "Integrity LOW" with no way to know if the assessment is current.

Two-part fix:
1. `LightweightMetricsRefresh` (called by govern every tick) updates the `*Last updated:*` timestamp in `docs/METRICS.md` each time it writes. Since it IS modifying the file (Completed Tasks + REVIEW_FAIL Rate), the timestamp should reflect that.
2. `arch status` parses the `*Last updated:*` timestamp from `docs/METRICS.md` and displays it alongside the Integrity Level, matching the age-annotation already used for the audit score line: `Report: ⚠ Integrity LOW (2h ago)`.
