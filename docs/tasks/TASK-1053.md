## Hansei
**Severity:** H1
**Category:** [SpecDrift]
**Decision:** The original `extractMetaClass`/`extractMetaSize` functions used a fragile regex on the `**Meta:**` line and fell back silently to defaults. This caused AC generation to be generic regardless of IDEA content. Fixed by introducing explicit `**Candidate-class:**` / `**Candidate-size:**` fields with strict validation and missing-field reporting. The `generateAll()` method now skips decided IDEAs and emits proposals only for undecided ones with all required fields.
**Constraint:** Existing IDEA files use the legacy format (Meta line or Estimated size section). Backward compat preserved: `extractCandidateSize` falls back to `## Estimated size` if `Candidate-size` is absent. Files that omit both get a missing-field report instead of a default.
**Cost:** Legacy IDEA files without `Candidate-class` will now show "missing candidate-class" in proposals instead of silently defaulting. Acceptable — the default was producing low-signal placeholder ACs anyway.
**Forward Action:** Existing `docs/refinement/IDEA-*.md` files could be updated to the new template format. No blocker — they all have Decision set so they are filtered out of proposals anyway.
