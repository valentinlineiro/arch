## TASK-1028: 90-day productization sprint — AC decomposition and sprint frame
**Meta:** P0 | L | READY | Focus:no | 6-writing | local | docs/

**Depends:** TASK-1026, TASK-1027

### Context

ARCH has a time-limited lead on the separation-of-powers insight for AI governance. The 90-day sprint has four locked objectives: (1) `arch init` that works in any repo in under 2 minutes, (2) `arch review` that finds real issues and suggests fixes, (3) `arch task capture` that feels like free value, (4) progressive CLI surface. This task produces the AC breakdown per objective that makes the sprint executable — it is a planning task, not an implementation task. Sub-tasks are created from the ACs this task produces. Source: IDEA-90-day-productization-sprint.

### Acceptance Criteria

  - `file: docs/90-day-sprint-acs.md`

  - `prose: docs/90-day-sprint-acs.md contains explicit out-of-scope note for existing-governance conflict resolution`

- [ ] ACs for objective 2 (`arch review`) include the `arch fix` companion command as a required deliverable; the exit action item must have a resolution path.
  - `prose: arch fix companion command appears as a required AC under objective 2`

  - `prose: meta line leakage is explicitly addressed in objective 3 ACs`

  - `prose: objective 4 ACs reference TASK-1027`
